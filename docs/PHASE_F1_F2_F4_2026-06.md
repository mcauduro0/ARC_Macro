# Phase F1 + F2 + F4 — the five autonomy screens (Holdout, Co-pilot, Risk, Research, Ledger)

The third frontend increment fills the ARC 2.0 nav: five real Mesa screens on the existing autonomy bridge,
plus a deepened Command. Built as a parallel batch — a background Workflow drafted the four read-only pages
(Holdout / Risk / Research / Ledger) concurrently, each grounded in the exact `@shared/autonomy` contract +
Mesa primitives + the honesty constraints, while the interactive Co-pilot workspace and the Command deepening
were hand-built. One authoritative `tsc` + vitest pass + a live bridge smoke validated the integration.

The honesty law holds on every screen: **pre-verdict, no forward Sharpe/DSR is ever rendered.** A realized
forward Sharpe/DSR appears in exactly one place — Holdout's `VerdictBlock`, reachable only from an
`if (verdict !== null)` branch (the holdout has fired). TypeScript's null-narrowing makes that structural.

## What was built

**Contract + wiring foundation:**
- `shared/autonomy.ts` — added the raw immutable ledger record types (`LedgerDecision`, `LedgerRealization`,
  `LedgerOperatorDecision`, `LedgerResponse`) mirroring `arc/autonomy/ledger.py` dataclasses exactly.
- `server/autonomyRouter.ts` — the `ledger` proxy route is now typed `LedgerResponse` (was `Record<…>`).
- `client/src/arc/useAutonomy.ts` — `useLedger(strategy)` hook (`trpc.autonomy.ledger.useQuery`).

**Co-pilot (F2) — the human-in-the-loop workspace** (`pages/CoPilot.tsx`, hand-built):
- Per sleeve: the current proposal (frozen vs proposed vs sized, VaR when the gate is active, circuit/drift
  flags), the **three operational streams** (frozen / live / operator — n, cum, max dd, last position), and
  the **decide control** (APPROVE / OVERRIDE / SKIP + rationale → `useDecide` → the bridge). A committed month
  renders locked (immutable per `(month, hash)`); a 409 from the ledger surfaces as a readable rejection.
- The invariant is stated and enforced: the buttons feed the **operator** stream only; the frozen holdout is
  untouchable and accrues deterministically toward the one-shot verdict.

**Holdout & Governance (F1)** (`pages/Holdout.tsx`):
- Per-sleeve + pool **pre-registration cards**: the deflation basis (n_trials = cumulative hypotheses,
  eval_at_n, dsr_min, forward_start research cutoff, booked/unbooked), the **verdict countdown** (AccrualBar +
  months-to-verdict + readiness message), and — only post-verdict — the verdict block (PASS/FAIL, DSR vs bar,
  realized forward Sharpe, n). The pool card carries the K_eff≈2.92 "~1y sooner IF several sleeves carry real
  edge (else pooling dilutes)" caveat. A footer explains the deflation logic.

**Risk (F3-front)** (`pages/Risk.tsx`):
- The pre-committed loss budget (RiskLimits: VaR 5.5%, ES 7.5%, α 0.05, 24-mo min, cornish-fisher), the
  per-sleeve **pre-trade gate** table (gate binding / VaR95 / ES / vol-target leverage / sized exposure /
  circuit dot — INACTIVE with `<24 mo` history), and an aggregated circuit/drift **flags** panel. Footer
  restates the invariant: the gate bounds losses only — risk control, never a signal, never the holdout.

**Research / Diagnostics (F4a)** (`pages/Research.tsx`):
- (a) live diagnostics off the proposals (action / gate / frozen-months / drift & circuit counts + a summary),
  (b) the documented **discipline & guardrails** (forward-only one-shot holdout, deflation, as-of/PIT data,
  leakage canaries, sandboxed operator overlay — labelled "documented method, not fresh results"), and
  (c) the three operational stream summaries. No fabricated IC/SHAP/null numbers — only live counts.

**Ledger / Audit (F4b)** (`pages/Ledger.tsx`):
- A sleeve selector (momentum / nowcast / fiscal) → the immutable record explorer over `useLedger`: decisions,
  realizations (per-stream filter, latest 24 + elision note), operator decisions — with hash provenance and the
  append-only / one-record-per-(month,stream) invariant stated up front.

**Command deepened** (`pages/Command.tsx`):
- An operational **stat strip** (operating / halted / warmup / gates active / operator decisions / awaiting
  decision) and a "● decide" nudge on rows whose current month is undecided.

**Routing** (`ArcApp.tsx`): `/co-pilot`, `/holdout`, `/risk`, `/research`, `/ledger` now render real screens;
`/macro` remains a frank Placeholder until F3-Macro (PR2).

## Process note (parallel build)
The four read-only pages were drafted by a background Workflow (4 agents, ~3.5 min, distinct files → zero merge
conflicts), each handed the exact contract + primitives + honesty rules inline and instructed to return a
structured self-review. Every draft self-reported `tsc` clean and the honesty gate; the orchestrator then ran
one authoritative `tsc` + vitest + a live smoke before integrating. The interactive Co-pilot + Command deepening
were hand-built in parallel with the drafting.
